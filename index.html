Your One Stop Device Manager

The revolutionary upgrades in IoT and the immense love towards losing cables in our recent technological trends has led to a massive outburst in the number of wireless devices we use. Sometimes, we know that it gets a little overwhelming for everyone. Don't worry though, we've got you. BlueRadar is a unified system that can manage, track and locate all the devices that you've paired with your Android smartphone.

Project Milestones

For the first Show and Tell, we plan on having the boilerplate of the application ready with the base MVVM architecture complete and communicating with one another. In addition to which, the data class to communicate with our database, a working navigation fragment and the base bluetooth functionality is imminent to completion, and our application will be able to retrieve all the paired BlueTooth devices.

For the second Show and Tell, we plan on having the bare minimum functionalities in a working state, i.e. Authentication, Tracking, Managing and Navigating. The connection between the appliaction and the mongoDB database will be functional. UI refinement for the bluetooth devices screen and the application in general. Implementation of multithreading will be working as well to ensure smooth lagless functionality.

For the final project, we plan on implementing the database, bluetooth and navigation implementations into a unified service, therefore reducing device runtime load and decreasing the battery use. We also plan on having the user profile feature implemented and map fragment that shows all the actively previously tracked devices in one place, so as to be able to pop any of them up on demand. Finally, migrating the back-end database from ROOM to MongoDB will be our final major implementation.

Once the course is complete (and we're done with our final exams), we plan on finishing up the bug fixes and cleaning up the application and create a .NET or REST API framework to receive APIs of the devices after user authentication is complete. This will give us a chance to have a web client which can readily be accessed even when the user's main device is not on hand. This is where we plan on publishing BlueRadar to the Google PlayStore.
